# Cache and Deploy Risk Register

| Risk                                 | Failure mode                                                                           | M0 mitigation                                                                                                      | Later gate                                                                                                                      |
| ------------------------------------ | -------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| Self-hosted cache failure            | The replacement cache is unavailable, slow, or serves incomplete objects.              | Keep current production deploy path unchanged; model cache-push and closure-prefill as separate phases.            | Local E2E tests for cache miss, broken object, and full closure prefill before any canary.                                      |
| Bad Nix commit on primary cache host | The host running cache or deploy-related services receives a broken system generation. | Document break-glass and rollback requirements without changing deploy behavior.                                   | Direct SSH rollback rehearsal independent of central controller and hosted cache.                                               |
| Bad deploy controller commit         | The controller cannot submit or reconcile deployments.                                 | Desired-state records include latest-only sequence and retry timestamps.                                           | Reconciler E2E tests for restart, retry, superseded deployment skip, and rollback.                                              |
| Broken substituters                  | Runners or targets cannot trust or reach configured caches.                            | Deployment inputs list substituters and trusted public keys explicitly.                                            | Preflight substituter checks that fail before activation and emit `closure-prefill` or `cache-push` errors.                     |
| Credential loss                      | Cache push, activation, signing, or target credentials are missing or revoked.         | Event schema has explicit command status and error details; inventory separates upload and activation credentials. | Credential rotation runbook and local tests for missing upload token, missing activation token, and invalid manifest signature. |
