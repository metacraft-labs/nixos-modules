#+TITLE: Shared Terraform Infrastructure

* Overview

  Company-agnostic Terraform/OpenTofu infrastructure that any consumer repo can
  adopt, so each repo does not duplicate CI workflows, policy scripts, dev-shell
  tooling, or bootstrap logic. The reusable pieces live here in nixos-modules;
  a consumer repo supplies only its own data (accounts, owners, zones, reviewed
  inventory) and thin-calls the shared modules and drivers.

  The shared agentic workflow is the same for every consumer: agents develop
  offline against mock providers, CI runs a credentialed plan and posts the
  result as a PR comment, a human approves, and CI applies on merge. Scheduled
  drift detection runs against the same backends.

* What is provided

  - *Reusable CI* — the reusable Terraform workflow
    (=.github/workflows/reusable-terraform-ci.yml=), the metadata-driven root
    discovery matrix (=terraform/ci/terraform-ci-matrix=), and the plan-policy
    scripts.
  - *Layer-0 bootstrap* — the AWS module (=terraform/aws/tf-bootstrap.nix=) and
    driver (=terraform/aws/aws-bootstrap=), and the CI-enabling GitHub module
    (=terraform/github/tf-bootstrap.nix=) and driver
    (=terraform/github/github-bootstrap=). See
    [[file:Terraform-Bootstrap-Runbook.md][the bootstrap runbook]].
  - *GitHub governance* — the governance engine
    (=terraform/github/governance.nix=), branch-protection rulesets
    (=terraform/github/branch-protection.nix=), and the import-phase tooling
    (=terraform/github/github-inventory=, =github-governance-import-blocks=,
    =github-governance-import-ci=).
  - *Cloudflare* — the read-only inventory tool
    (=terraform/cloudflare/cloudflare-inventory=); adoption sets stay per-repo.
  - *Methodology* — [[file:Terraform-Agent-Development-Methodology.md][development methodology]],
    [[file:Terraform-Testing.md][testing]], [[file:Terraform-Root-Layering.md][root layering]],
    [[file:Terraform-Import-Phase.md][the import/adoption phase]], and
    [[file:GitHub-Secret-Rotation.md][secret rotation]].

* Consuming a shared module

  A consumer repo's bootstrap root is a thin caller pinned to a nixos-modules
  revision; its Justfile invokes the shared drivers from that pinned source. See
  each module's README under =terraform/= and the bootstrap runbook for the
  end-to-end procedure. Deployment facts (accounts, owners, zones) live in the
  consumer's tracked Terraform/Nix files, never in this shared repo.
